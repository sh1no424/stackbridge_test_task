# Задание 1: Анализ требований
## Принятные допущения. 
* Так как получить ответы от заказчика или продукт менеджера невозможно. Для формирования исправленой версии тз, я принял допущения:
* Если цена в каталоге изменилась, она обновляется в корзине. Цена фиксируется только при успешном создании заказа.
* Удаление товара происходит по кнопке, а уменьшить количество товаров мешьше 1 нельзя. 
* Утро - с 06:00 до 10:00 Вечер с 18:00 до 20:00 Будние дни с пн по пт включительно. Часовой пояс Москвы. 
* Недоступные товары, должны отмечаться как недоступные. 
* Если товара не хватает, то в корзине должно отображаться доступное количество.

## Исправленная версия тз
| ID | Требование |
|---| --- |
| 01 | Пользователь может добавить в корзину товар доступный для заказа. |
| 02 | Каждый новый товар должен образовывать новую позицию в корзине. | 
| 03 | Система должна увеличивать количество единиц товара по позиции, если пользователь добавляет товар, который уже находиться в корзине. | 
| 04 | Итоговое количество единиц одного товара должно быть от 1 до 10 включительно. | 
| 05 | В корзине может одновременно находиться до 5 различных позиций товаров включительно. |
| 06 | Сумма всех единиц по всем позициям не должна быть больше 20. |
| 07 | При добавлении товара система должна контролировать: <br>1. Количество позиций в корзине <br>2. Сумму всех единиц товаров каждой позиции <br>3. Количество позиций в корзине | 
| 08 | Если операция добавления товара в корзину нарушает хотя бы одно условие, система должна отклонить операцию и оставить состояние корзины неизменным. | 
| 09 | Система должна уведомлять пользователя, если операция отклонена. Уведолмение должно содержать Код ошибки, сообщение:<br>1. ITEM_LIMIT - "Можно добавить не более 10 единиц одного товара" <br>2. POSITION_LIMIT - "Можно добавить максимум 5 различных товаров"<br>3. TOTAL_LIMIT - "Можно добавить не более 20 единиц товаров" | 
| 10 | Пользователь должен иметь возможность менять целое количество позиции от 1 до 10 включительно. |
| 11 | При изменении пользователем количества товаров, система должна проверять ограничения. |
| 12 | Если количество единц товара равно 1, то уменьшение должно быть недоступно. |
| 13 | Пользователь может удалить товар с помощью кнопки удалить |
| 14 | После удаления товар не должен отображаться в корзине и учитываться в стоимости. |
| 15 | Система должна получать актуальную цену из каталога, при добавлении товара в корзину. |
| 16 | Система должна обновлять цены товаров в корзине, если цена в каталоге на них изменилась. |
| 17 | При успешном создании заказа, система должна фиксировать окончательную цену. |
| 18 | Если товар недоступен после добавления в корзину, система должна отметить позицию как "недоступно". |
| 19 | Если остаток товаров меньше количества товара в корзине, система должна отображать доступное количество. |
| 20 | Система должна запретить создание заказа, если некоторые позиции корзины недоступны. |
| 21 | На странице корзины должна отображаться реклама других продуктов. | 
| 22 | Реклама должна отображаться с 06:00 до 10:00 и с 18:00 до 22:00 по московскому времени. С понедельника по пятницу включительно | 
| 23 | Для каждой позиции корзины система должна отображать поля: id товара, наименование товара, изображение, количество, цену за единицу, общую стоимость, статус доступности. |
| 24 | Система должна отображать общую стоимость всех позиций на странице корзины. |
